\section{Reproducing the baseline}
\label{sec:reproduce}

\paragraph{Run provenance.}
We ran \texttt{scripts/train.py} unmodified at baseline commit
\href{https://github.com/borisbolliet/tpu-2026/tree/324abbe4b4e229ea812223856393547db4fbb53e}{\texttt{324abbe}}
with the default \texttt{config.py}, on a single Cloud TPU \texttt{v6e-1} (1 chip,
$32$\,GiB HBM). The run completed all \texttt{MAX\_STEPS}$=3364$ GRPO steps (the
$90\%$ train split of $3738$ prompts; $6728$ rollouts at $K{=}2$) in
$\approx\!4.7$\,h wall-clock ($16{,}999$\,s), logged to W\&B run
\href{https://wandb.ai/sichengma0514-university-of-cambridge/a8-grpo/runs/bdbugenj}{\texttt{bdbugenj}}.
Defaults: $\beta{=}0.08$, $\varepsilon{=}0.2$, rank${=}\alpha{=}64$, LR
$3{\times}10^{-6}$ (cosine, $10\%$ warmup), grad clip $0.1$, $K{=}2$, rollout
$T{=}0.9$, $\le768$ tokens.

\noindent The baseline does \emph{not} reproduce as an improvement. The base
model already solves $51.6\%$ (its answers parse, rarely in the strict template),
whereas every saved checkpoint is worse, the ``resulting'' step-$3364$ one
collapsing to $3.1\%$. Tellingly, step $2000$ has the \emph{highest} format
accuracy yet lower correctness than base: format shaping is optimised at the
expense of the arithmetic (reward hacking) before the run destabilises entirely
(Fig.~\ref{fig:baseline-curves}).

\paragraph{Training curves.}
The reward (Fig.~\ref{fig:baseline-curves}, left; $\bar r$ summed over the four
reward functions, max $10$) rises fast --- the template is acquired within
$\sim$150 steps, peaking at $\bar r\approx 6$--$7$ near step $450$ --- then
\emph{declines steadily} for the rest of training, reaching
$\bar r\approx 0$ (often negative) by step $\sim$2700 with no recovery. The
held-out (val) reward, logged every $64$ steps, tracks training closely: genuine
policy degradation, not over-fitting. KL stays small ($\lesssim 0.7$) for most of
the run but spikes (up to $\approx 41$, inset) in the final third --- unstable
updates; every surviving checkpoint lies in this post-peak regime
(Table~\ref{tab:baseline-acc}), which Sec.~\ref{sec:improve} targets.

\paragraph{Baseline patches.}
Training ran as-shipped (tree $=$ \texttt{324abbe} up to a two-line path edit in
the \texttt{run\_tmux.sh} launcher; algorithm and \texttt{config.py}
\emph{unchanged}). Three fixes, all outside the training loop: \textbf{(1) W\&B routing} ---
\texttt{config.py} defaults to another account; we set \texttt{WANDB\_ENTITY}
and \texttt{WANDB\_PROJECT} via environment;
\textbf{(2) evaluation restore} --- as shipped, \texttt{evaluate.py} never loads
a checkpoint, and its freshly-built adapter is a no-op
($\Delta W{=}\tfrac{\alpha}{r}BA$ with $B{=}0$), so it only ever scored the
\emph{base} model; we added \texttt{-{}-ckpt-dir/-{}-steps} flags that restore
the adapter via Tunix's \texttt{CheckpointManager} (as in \texttt{chat.py}),
default invocation unchanged; \textbf{(3) dataset re-read} --- re-opening the
cached TFDS dataset in a fresh process fails under \texttt{protobuf}~7.34
(\texttt{FieldDescriptor} lost \texttt{.label} in $\ge$5);
deleting the prepared \texttt{data/.../gsm8k/1.0.0} lets \texttt{evaluate.py}
re-prepare in-process from the cached raw download --- training's own code path,
\emph{no code change}.

\section{Understanding the baseline}
\label{sec:understand}

% TODO (a): what plays the role of pi_theta, pi_old, pi_ref in the LoRA setup,
% and which of them carries trainable parameters?
\paragraph{(a) $\pi_\theta$, $\pi_{\mathrm{old}}$, $\pi_{\mathrm{ref}}$ in the LoRA setup.}
In the LoRA setup, the $\pi_\theta$ corresponds to the current policy with the LoRA adapter applied, this is the one with trainable parameters; 
$\pi_{\mathrm{old}}$ is the behaviour policy from which rollouts are sampled, which in Tunix's GRPO implementation is a periodically updated copy of $\pi_\theta$ (the policy before the current update);
and $\pi_{\mathrm{ref}}$ is the fixed base model \texttt{gemma-3-1b-it} without any LoRA adapter.

% TODO (b): group size K in config.py, and which advantage estimator Tunix
% uses -- standard group baseline or leave-one-out?
\paragraph{(b) Group size $K$ and the advantage estimator.}
$K=2$: \texttt{NUM\_GENERATIONS} in \texttt{config.py}, passed as
\texttt{GRPOConfig(num\_generations$=$2)} in \texttt{train.py}; each prompt is rolled out
twice, the pair forming one group. Tunix resolves the estimator from a registry:
\texttt{advantage\_estimator} defaults to \texttt{"grpo"} (not overridden), selecting
\texttt{compute\_advantages} in \texttt{tunix/rl/algo\_core.py} --- the \emph{standard}
group baseline $\hat A_i=(r_i-\bar r)/(\sigma_r+10^{-6})$ with the group's sample std
(\texttt{ddof}$=$1), \emph{not} leave-one-out (RLOO ships in the same file under the
separate name \texttt{"rloo"}). With $K{=}2$, $r_i-\bar r=\pm\tfrac12(r_1-r_2)$ and
$\sigma_r=|r_1-r_2|/\sqrt2$: whenever the rewards differ, $\hat A_i=\pm1/\sqrt2$
\emph{regardless of margin}; ties give $\hat A_i\approx0$.

% TODO (c): in rewards.py, which terms are shaping rewards and which is the
% "true" correctness reward? In terms of within-group reward variance sigma_r,
% what happens early in training with the correctness reward alone?
\paragraph{(c) Shaping vs.\ correctness rewards and $\sigma_r$.}
\texttt{match\_format\_exactly} ($+3$ for a full template match) and
\texttt{match\_format\_approximately} ($\pm0.5$ per tag, range $[-2.5,2.5]$) are pure
\emph{shaping} rewards: they score the output format, never the mathematics. The ``true''
correctness reward is \texttt{check\_answer}: it extracts the \texttt{<answer>} field via the
full-template regex and grades the number ($+3$ exact, $+1.5$ after stripping, $+0.5/{+}0.25$
within $10/20\%$, $-1$ wrong, $-0.5$ unparseable). \texttt{check\_numbers} ($+1.5$ iff
\emph{any} number after \texttt{<answer>} equals the gold answer) is a binary correctness
\emph{fallback}, itself shaping towards an extractable number. With the
correctness reward alone, early in training the per-rollout success probability $p$ is tiny
(doubly so for \texttt{check\_answer}, gated on template parsing), so both
rollouts of a group almost surely receive the \emph{same} reward: a group is informative only
when the two disagree, with probability $2p(1-p)\approx2p\to0$ (for a binary reward of size
$c$, $\mathbb{E}[\sigma_r^2]=c^2p(1-p)$). Every degenerate group has $r_i=\bar r$, hence
$\hat A_i=0$ and a zero policy-gradient term. The graded format
terms already differ between two \emph{wrong} answers, keeping $\sigma_r>0$ (and, by (b),
$|\hat A_i|=1/\sqrt2$) from the first steps --- letting training start, and later letting
format be optimised at correctness's expense (Sec.~\ref{sec:reproduce}).

% TODO (d): where is the clipped (PPO-style) surrogate implemented in Tunix,
% and what does the clip range epsilon correspond to in config.py? Note any
% deviation from the lecture form (e.g. per-token vs per-sequence ratio).
\paragraph{(d) The clipped surrogate and $\varepsilon$.}
The clip is implemented in \texttt{grpo\_loss\_fn}, the registered \texttt{"grpo"}
policy loss in \texttt{tunix/rl/algo\_core.py}: per-token log-ratios are exponentiated to
$\rho_t$ and the per-token loss is
$\max\bigl(-\hat A\,\rho_t,\,-\hat A\,\mathrm{clip}_{1\pm\varepsilon}(\rho_t)\bigr)$,
the lecture objective with $\varepsilon=\texttt{EPSILON}=0.2$ from \texttt{config.py} (via
\texttt{GRPOConfig(epsilon$=$0.2)}). Two deviations from the lecture form. \emph{(1)
Per-token ratio:} $\rho_t$ is computed per token while $\hat A_i$ is per-sequence, broadcast
to every completion token, so clipping acts token-wise. \emph{(2) The clip is inert here:}
with $\mu=\texttt{NUM\_ITERATIONS}=1$ Tunix computes no separate $\pi_{\mathrm{old}}$
log-probs, substituting \texttt{stop\_gradient} of the current ones: $\rho_t\equiv1$ in
the forward pass, the clip never binds (\texttt{pg\_clipfrac}{=}0), and the surrogate
reduces to group-relative REINFORCE $-\hat A_i\nabla_\theta\log\pi_\theta$. The KL anchor is a separate additive penalty with $\beta=0.08$, estimated per token by
the naive $\log\pi_\theta-\log\pi_{\mathrm{ref}}$ \citep{schulman2020kl} ---
sign-indefinite, and indeed ${\sim}1.5\%$ of logged steps dip below zero.

\paragraph{Motivation.}
Sec.~\ref{sec:reproduce} ends in reward-hacking collapse (I.2c). We target it with the
two levers the lectures expose, laid out as a $2{\times}2$ over (group size $K$, KL weight
$\beta$) plus two $K{=}2$ probes. \emph{(i)~Effective sample size (primary).} At $K{=}2$ the
group-normalised advantage is degenerate-coarse --- $\hat A_i\in\{0,\pm1/\sqrt2\}$ whatever
the margin (I.2b) --- and the estimator variance scales as $1/K_{\mathrm{eff}}$ with
$K_{\mathrm{eff}}\!\propto\!K$ (I.4~Eq.~\eqref{eq:q1-keff}). Prediction: raising
$K{:}\,2{\to}8$ grades the advantage and lowers gradient variance, stabilising training.
\emph{(ii)~The KL trust region (secondary).} $\beta\,\mathrm{KL}(\pi_\theta\|\pi_{\mathrm{ref}})$
should bound drift and, with it, reward hacking
\citep{schulman2015trpo,shao2024deepseekmath}; we sweep $\beta\in\{0,0.08,0.3\}$ (R1 raises
it, R4/R3b switch it off). A reward probe R2 adds DAPO's soft over-long penalty
\citep[Eq.~13]{yu2025dapo} ($0$ ramping to $-1$ over $1500$--$3000$ chars).

\paragraph{Discussion.}
\emph{Group size is the lever.} At matched $\beta{=}0.08$, raising $K{:}\,2{\to}8$ turns
collapse into a win --- R0 craters to $6.2\%$ while R5 holds $55.6\%$; at $\beta{=}0$ it lifts
R4's $48.6\%$ to R3b's $\mathbf{57.2\%}$, whose CI $[54.6,59.9]$ is \emph{disjoint} from base
$[44.7,50.0]$: a real $+9.8$-pt gain, not the parity a $K{=}2$/$N{=}64$ read would suggest.
\emph{Mechanism (theory).} The logged within-group advantage std is $0.935{=}\sqrt{(K{-}1)/K}$
at $K{=}8$ (Eq.~\eqref{eq:q1-vara}; vs $1/\sqrt2$ at $K{=}2$, I.2b), so the group is graded over
$8$ ranks not $3$ values and $K_{\mathrm{eff}}$ quadruples (Eq.~\eqref{eq:q1-keff}); the
lower-variance gradient shows in KL (at matched $\beta$, $K{=}8$ pins $\mathrm{KL}{\approx}0.2$
vs $K{=}2$'s drift past $4$, Fig.~\ref{fig:improve-curves}) and in the decomposition
(\texttt{check\_answer} $+1.4$ at $K{=}8$ vs $-0.38$ at $K{=}2$, Fig.~\ref{fig:diagnostic}):
correctness is no longer traded for shaping. \emph{$\beta$ --- honest negatives.} The KL-leash
prediction was contradicted: at $K{=}2$, \emph{more} leash drifted \emph{more}, R1$(\beta{=}0.3)$
collapsing hardest (reward $-2.5$, KL${\to}33$) --- with a misspecified reward (I.2c) and the
sign-indefinite KL estimator (I.2d) a large $\beta$ only amplifies gradient noise. Once $K{=}8$,
$\beta$ barely matters (R5${\approx}$R3b). \emph{Caveat:} one seed per cell, and the gain costs
${\sim}4{\times}$ rollout compute --- the $K_{\mathrm{eff}}$/compute trade-off of
Eq.~\eqref{eq:q1-keff}.

\subsection{Variance of the group-normalised gradient estimator}
\label{sec:theory-q1}
For a fixed prompt $q$ and $K\ge 2$ responses $a_1,\dots,a_K$ (treated as fixed),
let $r_1,\dots,r_K$ be i.i.d.\ rewards with mean $\mu$ and variance $\sigma^2$.
Define the group mean and normalised advantage
\begin{equation}
  \bar r \;=\; \frac{1}{K}\sum_{i=1}^{K} r_i,
  \qquad
  \hat A_i \;=\; \frac{r_i-\bar r}{\sigma_r},
  \label{eq:q1-adv}
\end{equation}
treating $\sigma_r=\sigma$ as a known constant. With deterministic score vectors
$h_i=\nabla_\theta\log\pi_\theta(a_i\mid q)\in\mathbb{R}^d$, define
\begin{equation}
  X_i \;=\; h_i\,\hat A_i,
  \qquad
  \hat g \;=\; \frac{1}{K}\sum_{i=1}^{K} X_i \;\in\; \mathbb{R}^d.
  \label{eq:q1-ghat}
\end{equation}

\paragraph{(i) $\mathbb{E}[X_i]$, $\mathbb{E}[\hat g]$, and why the true gradient need not vanish. \hfill\textnormal{[2]}}
Since each $h_i$ is deterministic, $\mathbb{E}[X_i]=h_i\,\mathbb{E}[\hat A_i]$, and the
normalised advantage is mean-zero:
\begin{equation}
  \mathbb{E}[\hat A_i]
  \;=\; \frac{\mathbb{E}[r_i]-\mathbb{E}[\bar r]}{\sigma_r}
  \;=\; \frac{\mu-\mu}{\sigma_r}
  \;=\; 0 ,
  \label{eq:q1-meanzero}
\end{equation}
because $\mathbb{E}[r_i]=\mathbb{E}[\bar r]=\mu$. Hence $\mathbb{E}[X_i]=0$ and
$\mathbb{E}[\hat g]=\tfrac{1}{K}\sum_{i=1}^{K}\mathbb{E}[X_i]=0$.
This does not imply the \emph{true} policy gradient vanishes. Here $\mathbb{E}[\hat g]=0$ holds
\emph{by construction}: the responses $a_i$ (hence the scores $h_i$) are held fixed, and the
group-centred advantage makes $\mathbb{E}[\hat A_i]=0$ termwise. The genuine policy gradient is
instead an expectation over actions \emph{drawn from the policy},
$\nabla_\theta J=\mathbb{E}_{a\sim\pi_\theta(\cdot\mid q)}[\nabla_\theta\log\pi_\theta(a\mid q)\,A(a)]$,
which need not vanish. Indeed, removing the centring leaves
$\mathbb{E}\bigl[\tfrac{1}{K}\sum_i h_i r_i\bigr]=\mu\,\bar h$, generically nonzero
(with $\bar h=\tfrac{1}{K}\sum_i h_i$): it is the baseline subtraction under fixed responses,
not a stationary policy, that drives $\mathbb{E}[\hat g]$ to~$0$.

\paragraph{(ii) $\mathrm{Cov}(X_i,X_j)$ for $i\neq j$. \hfill\textnormal{[6]}}
Since $\mathbb{E}[X_i]=0$ (part i), the cross-covariance equals the centred second moment, and
the deterministic scores factor out of the expectation:
\begin{equation}
  \mathrm{Cov}(X_i,X_j)=\mathbb{E}[X_iX_j^{\top}]=h_ih_j^{\top}\,\mathbb{E}[\hat A_i\hat A_j],
  \label{eq:q1-cov-factor}
\end{equation}
so the problem reduces to the scalar $\mathbb{E}[\hat A_i\hat A_j]$. Substituting
$\hat A_k=(r_k-\bar r)/\sigma_r$ (with $\mathbb{E}[r_k-\bar r]=0$, this is again a covariance),
the shared mean $\bar r$ couples the two factors \emph{even though} $r_i\perp r_j$; expanding by
bilinearity, with $\mathrm{Cov}(r_i,r_j)=\sigma^2\delta_{ij}$ and
$\mathrm{Cov}(r_i,\bar r)=\mathrm{Var}(\bar r)=\sigma^2/K$,
\begin{align*}
  \mathbb{E}[\hat A_i\hat A_j]
    &= \frac{1}{\sigma_r^{2}}\,\mathrm{Cov}(r_i-\bar r,\;r_j-\bar r) \\
    &= \frac{1}{\sigma_r^{2}}\Bigl[\,\underbrace{\mathrm{Cov}(r_i,r_j)}_{0\ (i\neq j)}
       -\mathrm{Cov}(r_i,\bar r)-\mathrm{Cov}(\bar r,r_j)+\mathrm{Var}(\bar r)\Bigr] \\
    &= \frac{1}{\sigma_r^{2}}\left(0-\frac{\sigma^2}{K}-\frac{\sigma^2}{K}+\frac{\sigma^2}{K}\right)
     = -\frac{\sigma^2}{K\sigma_r^{2}}.
\end{align*}
Hence, for $i\neq j$,
\begin{equation}
  \mathrm{Cov}(X_i,X_j)=-\frac{\sigma^2}{K\sigma_r^{2}}\,h_ih_j^{\top}
  \;=\;-\frac{1}{K}\,h_ih_j^{\top}\quad(\sigma_r=\sigma).
  \label{eq:q1-cov}
\end{equation}
The scalar prefactor is \emph{negative}, so distinct normalised advantages are negatively
correlated. The coupling runs entirely through the shared group mean: a reward above $\bar r$
inflates the mean and pushes every other centred advantage down. Equivalently
$\sum_i\hat A_i\equiv0$, so the advantages cannot all move together --- the off-diagonal
covariances must be negative to offset the diagonal variances.

\paragraph{(iii) $\mathrm{Var}(\hat g)$ and the effective sample size $K_{\mathrm{eff}}$. \hfill\textnormal{[7]}}
First we expand the variance by definition:
\begin{equation}
  \mathrm{Var}(\hat g)=\frac{1}{K^2}\Bigl[\sum_{i}\mathrm{Var}(X_i)+\sum_{i\neq j}\mathrm{Cov}(X_i,X_j)\Bigr].
  \label{eq:q1-varg}
\end{equation}
The off-diagonal terms are Eq.~\eqref{eq:q1-cov}; the diagonal is
$\mathrm{Var}(X_i)=\mathrm{Var}(\hat A_i)\,h_ih_i^{\top}$ with
\begin{equation}
  \mathrm{Var}(\hat A_i)=\frac{\mathrm{Var}(r_i-\bar r)}{\sigma_r^{2}}
  =\frac{\sigma^2-2\sigma^2/K+\sigma^2/K}{\sigma_r^{2}}
  =\frac{K-1}{K}\,\frac{\sigma^2}{\sigma_r^{2}}.
  \label{eq:q1-vara}
\end{equation}
Write $S=\sum_i h_ih_i^{\top}$ and $\bar h=\tfrac1K\sum_i h_i$, so that
$\sum_{i\neq j}h_ih_j^{\top}=K^2\bar h\bar h^{\top}-S$. The off-diagonal $-\tfrac1K$ exactly
fills the diagonal deficit $\tfrac{K-1}{K}$, so the coefficient of $S$ collapses to
$\sigma^2/\sigma_r^{2}$:
\begin{align}
  \mathrm{Var}(\hat g)
    &=\frac{1}{K^2}\Bigl[\frac{K-1}{K}\frac{\sigma^2}{\sigma_r^{2}}\,S
       -\frac{\sigma^2}{K\sigma_r^{2}}\bigl(K^2\bar h\bar h^{\top}-S\bigr)\Bigr]\nonumber\\
    &=\frac{\sigma^2}{K^2\sigma_r^{2}}\bigl(S-K\bar h\bar h^{\top}\bigr)
     =\frac{\sigma^2}{K^2\sigma_r^{2}}\sum_{i=1}^{K}(h_i-\bar h)(h_i-\bar h)^{\top}.
  \label{eq:q1-varg-result}
\end{align}

\noindent\textbf{Comparison.} The naive i.i.d.\ guess $\tfrac1K\mathrm{Var}(X_1)$ keeps the raw
outer product $h_1h_1^{\top}$, whereas the true variance depends only on the \emph{centred}
scores $h_i-\bar h$: the group baseline removes the common component $\bar h$ from every score,
leaving only their spread about the mean.

\smallskip\noindent\textbf{Effective sample size.} Matching the i.i.d.\ form
$\mathrm{Var}(\hat g)=\tfrac{1}{K_{\mathrm{eff}}}\mathrm{Var}(X_1)$ via the total variance defines
\begin{equation}
  K_{\mathrm{eff}}:=\frac{\mathrm{tr}\,\mathrm{Var}(X_1)}{\mathrm{tr}\,\mathrm{Var}(\hat g)}
  =\frac{K(K-1)\,\lVert h_1\rVert^2}{\sum_{i}\lVert h_i-\bar h\rVert^2},
  \label{eq:q1-keff}
\end{equation}
the number of independent draws of $X_1$ that would reproduce $\hat g$'s variance (the scale
$\sigma^2/\sigma_r^{2}$ cancels).

\smallskip\noindent\textbf{Limits.} \emph{(i) Large $K$:} if the scores stay spread so that
$\sum_i\lVert h_i-\bar h\rVert^2=\Theta(K)$, then $\mathrm{Var}(\hat g)=\Theta(1/K)$ and
$K_{\mathrm{eff}}=\Theta(K)$. \emph{(ii) Aligned scores}
($h_i\equiv h$): $h_i-\bar h=0$, so $\mathrm{Var}(\hat g)=0$ and 
$\hat g=h\,\tfrac1K\sum_i\hat A_i\equiv0$ because $\sum_i\hat A_i\equiv0$. Formally
$K_{\mathrm{eff}}\to\infty$ which is degeneracy since identical rollouts carry
\emph{no} gradient signal; this is the same vanishing-update failure as an all-equal-reward group in
I.2(c).

\smallskip\noindent\textbf{Empirical check (Sec.~\ref{sec:improve}).} The $K$-sweep there
bears this out: the logged within-group advantage std equals $\sqrt{(K{-}1)/K}$ exactly
($0.935$ at $K{=}8$ vs $1/\sqrt2{\approx}0.707$ at $K{=}2$), confirming
$\mathrm{Var}(\hat A_i)=\tfrac{K-1}{K}$ from Eq.~\eqref{eq:q1-vara}, and raising $K{:}\,2{\to}8$
(larger $K_{\mathrm{eff}}$) is what converts the baseline's reward-hacking collapse into a
held-out improvement (Table~\ref{tab:improve-acc}).

\subsection{KL-regularised policy improvement and the clipped surrogate}
\label{sec:theory-q2}
Let $\mathcal{A}$ be an action space and $\pi_t$ the policy at iteration $t$, with
a fixed advantage function $A_t:\mathcal{A}\to\mathbb{R}$. For policies $p,q$,
$\mathrm{KL}(p\|q)=\sum_a p(a)\log\frac{p(a)}{q(a)}$ (with $0\log 0=0$).
Consider the regularised update
\begin{equation}
  \pi_{t+1} \;=\; \arg\max_{\pi}\;
  \Bigl\{\, \mathbb{E}_{a\sim\pi}[A_t(a)] \;-\; \tfrac{1}{\eta}\,\mathrm{KL}(\pi\|\pi_t) \,\Bigr\},
  \qquad \eta>0.
  \label{eq:q2-update}
\end{equation}

\paragraph{(i) Unique optimiser of \eqref{eq:q2-update} and the role of $\eta$. \hfill\textnormal{[3]}}
This is a constrained optimisation problem with a Lagrange multiplier $\lambda$ for $\sum_a\pi(a)=1$.
\begin{equation}
  \mathcal{L}(\pi,\lambda)=\sum_a\pi(a)A_t(a)-\frac1\eta\sum_a\pi(a)\log\frac{\pi(a)}{\pi_t(a)}
  -\lambda\Bigl(\sum_a\pi(a)-1\Bigr).
  \label{eq:q2-lagrangian}
\end{equation}
By taking the derivative with respect to $\pi(a)$, we have $\partial_{\pi(a)}[\pi\log(\pi/\pi_t)]=\log(\pi(a)/\pi_t(a))+1$.
This gives the stationarity condition
\begin{equation}
  A_t(a)-\frac1\eta\Bigl(\log\frac{\pi(a)}{\pi_t(a)}+1\Bigr)-\lambda=0
  \quad\Longrightarrow\quad
  \pi(a)\propto\pi_t(a)\,e^{\eta A_t(a)}.
  \label{eq:q2-stationarity}
\end{equation}
Incorporating the normalising constant $Z=\sum_{a'}\pi_t(a')\,e^{\eta A_t(a')}$ gives the unique optimiser:
\begin{equation}
  \pi_{t+1}(a)=\frac{\pi_t(a)\,e^{\eta A_t(a)}}{\sum_{a'}\pi_t(a')\,e^{\eta A_t(a')}}
  =\frac1Z\,\pi_t(a)\,e^{\eta A_t(a)}.
  \label{eq:q2-softmax}
\end{equation}
The role of $\eta$ as a step-size parameter could be shown by expanding \eqref{eq:q2-softmax} to first order in $\eta$:
\[
  \pi_{t+1}(a)\approx\pi_t(a)\bigl[1+\eta\bigl(A_t(a)-\mathbb{E}_{\pi_t}[A_t]\bigr)\bigr],
\]
i.e. $\pi_{t+1}-\pi_t \approx \eta\,\pi_t\,(A_t-\mathbb{E}_{\pi_t}[A_t])$, which is a small update in the direction of the advantage function, scaled by $\eta$.

\paragraph{(ii) Monotone improvement and where it fails. \hfill\textnormal{[5]}}
We start by substituting the exact advantage to the performance of $\pi_t$
\begin{equation}
  J(\pi_t)=\mathbb{E}_{a\sim\pi_t}\!\bigl[A^{\pi_t}(a)\bigr]=0 .
  \label{eq:q2-baseline}
\end{equation}
Since $\pi_{t+1}$ maximises \eqref{eq:q2-update}, its value beats any $\pi$;
We then evaluate that bound at $\pi=\pi_t$, where the KL term vanishes.
\begin{equation}
  \mathbb{E}_{a\sim\pi_{t+1}}\!\bigl[A_t(a)\bigr]-\frac1\eta\mathrm{KL}(\pi_{t+1}\|\pi_t)
  \;\ge\;
  \mathbb{E}_{a\sim\pi_t}\!\bigl[A_t(a)\bigr]-\frac1\eta\underbrace{\mathrm{KL}(\pi_t\|\pi_t)}_{=0}
  \;=\;0 .
  \label{eq:q2-opt}
\end{equation}
Rearrange the terms to get a lower bound on the performance of $\pi_{t+1}$, which is non-negative, hence 
monotone improvement of the performance of $\pi_t$ (equality holds iff $\pi_{t+1}=\pi_t$)
\begin{equation}
  J(\pi_{t+1})=\mathbb{E}_{a\sim\pi_{t+1}}\!\bigl[A_t(a)\bigr]
  \;\ge\;\frac1\eta\mathrm{KL}(\pi_{t+1}\|\pi_t)\;\ge\;0=J(\pi_t).
  \label{eq:q2-monotone}
\end{equation}
This guarantee hinges on the \emph{exact} advantage $A_t=A^{\pi_t}$: that is what identifies
$\mathbb{E}_{a\sim\pi_{t+1}}[A_t]$ with $J(\pi_{t+1})$ in \eqref{eq:q2-monotone}. With an estimated or
stale $\hat A_t\neq A^{\pi_t}$, \eqref{eq:q2-opt} only certifies improvement of the \emph{surrogate}
$\mathbb{E}[\hat A_t]$, while the true performance carries an extra bias
\begin{equation}
  J(\pi_{t+1})=\mathbb{E}_{a\sim\pi_{t+1}}\!\bigl[\hat A_t\bigr]
  +\mathbb{E}_{a\sim\pi_{t+1}}\!\bigl[A^{\pi_t}-\hat A_t\bigr],
  \label{eq:q2-bias}
\end{equation}
whose second term can be negative, so $J(\pi_{t+1})\ge J(\pi_t)$ no longer holds. (Separately, with a
neural-network $\pi_\theta$ or a noisy finite-sample KL, $\pi_{t+1}$ is not the exact maximiser and
\eqref{eq:q2-opt} itself fails.)

\paragraph{(iii) Trust region of the clipped surrogate vs.\ the KL constraint. \hfill\textnormal{[2]}}
With $\pi_{\mathrm{old}}=\pi_t$, $a_t\sim\pi_{\mathrm{old}}(\cdot\mid q)$,
\begin{equation}
  \rho_t(\theta)=\frac{\pi_\theta(a_t\mid q)}{\pi_{\mathrm{old}}(a_t\mid q)},
  \qquad
  J^{\mathrm{clip}}(\theta)=\mathbb{E}_t\!\Bigl[\min\bigl(\rho_t(\theta)\hat A_t,\;
  \mathrm{clip}(\rho_t(\theta),1-\varepsilon,1+\varepsilon)\hat A_t\bigr)\Bigr].
  \label{eq:q2-clip}
\end{equation}
The clip caps the surrogate as soon as a sample's ratio leaves the band (in the objective-improving
direction, through the $\min$), freezing its gradient, so the trust region it enforces is a per-sample
box on the likelihood ratio
\begin{equation}
  \rho_t(\theta)\in[\,1-\varepsilon,\;1+\varepsilon\,]\qquad\text{(one box per sampled }a_t).
  \label{eq:q2-clipband}
\end{equation}
This contrasts with the single aggregate constraint implicit in \eqref{eq:q2-update},
\begin{equation}
  \mathrm{KL}\bigl(\pi_\theta(\cdot\mid q)\,\|\,\pi_{\mathrm{old}}(\cdot\mid q)\bigr)
  =\sum_a \pi_\theta(a\mid q)\log\frac{\pi_\theta(a\mid q)}{\pi_{\mathrm{old}}(a\mid q)}\;\le\;\delta ,
  \label{eq:q2-klball}
\end{equation}
which couples all actions and weights each by its probability mass. The qualitative difference is that
the clip acts coordinate-wise on each sampled action and is blind to probability mass, so many in-band
moves can still accumulate into a large total KL, whereas \eqref{eq:q2-klball} bounds that aggregate
divergence directly. Outside the band the clip gives zero gradient hence no restoring pull back
towards $\pi_{\mathrm{old}}$, whereas the KL penalty grows with the deviation.

\subsection{Mixture-proposal sampling, importance correction, and weight collapse}
\label{sec:theory-q3}
Fix $q$ and a fixed sampling policy $\pi_{\mathrm{old}}(\cdot\mid q)$; let
$R(q,a)\in\mathbb{R}$ be a deterministic reward and
$V^{\pi}(q)=\mathbb{E}_{a\sim\pi(\cdot\mid q)}[R(q,a)]$. Standard GRPO draws
$a_1,\dots,a_K\stackrel{\mathrm{iid}}{\sim}\pi_{\mathrm{old}}$, sets $r_i=R(q,a_i)$ and forms
\begin{equation}
  \hat g_{\mathrm{GRPO}} \;=\; \frac{1}{K}\sum_{i=1}^{K}\nabla_\theta\log\pi_\theta(a_i\mid q)\,\hat A_i,
  \qquad \hat A_i=\frac{r_i-\bar r}{\sigma_r}.
  \label{eq:q3-grpo}
\end{equation}
Consider instead the mixture proposal
\begin{equation}
  \pi_{\mathrm{mix}}(a\mid q) \;=\; (1-\alpha)\,\pi_{\mathrm{old}}(a\mid q) \;+\; \alpha\,\pi_{\mathrm{unif}}(a\mid q),
  \qquad \alpha\in[0,1),
  \label{eq:q3-mix}
\end{equation}
with $\pi_{\mathrm{unif}}$ uniform over responses.

\paragraph{(i) Importance-corrected estimator $\hat g_{\mathrm{mix}}$. \hfill\textnormal{[3]}}
We start from the standard policy gradient, whose expectation is over the old policy $\pi_{\mathrm{old}}$,
and change measure to the mixture $\pi_{\mathrm{mix}}$ by importance sampling:
\begin{equation}
\begin{aligned}
  g&=\mathbb{E}_{a\sim\pi_{\mathrm{old}}}\!\bigl[\nabla_\theta\log\pi_\theta(a\mid q)\,\hat A\bigr]\\
   &=\mathbb{E}_{a\sim\pi_{\mathrm{mix}}}\!\Bigl[\tfrac{\pi_{\mathrm{old}}(a\mid q)}{\pi_{\mathrm{mix}}(a\mid q)}\,\nabla_\theta\log\pi_\theta(a\mid q)\,\hat A\Bigr].
\end{aligned}
\label{eq:q3-is}
\end{equation}
The Monte-Carlo estimate of \eqref{eq:q3-is} with $K$ draws from $\pi_{\mathrm{mix}}$ gives
\begin{equation}
  \hat g_{\mathrm{mix}}=\frac1K\sum_{i=1}^K w_i\,\nabla_\theta\log\pi_\theta(a_i\mid q)\,\hat A_i,
  \qquad a_i\sim\pi_{\mathrm{mix}}.
  \label{eq:q3-gmix}
\end{equation}
Substituting the mixture \eqref{eq:q3-mix} into $w_i=\pi_{\mathrm{old}}/\pi_{\mathrm{mix}}$ gives the explicit weight:
\begin{equation}
  w_i=\frac{\pi_{\mathrm{old}}(a_i\mid q)}{\pi_{\mathrm{mix}}(a_i\mid q)}
     =\frac{\pi_{\mathrm{old}}(a_i\mid q)}{(1-\alpha)\,\pi_{\mathrm{old}}(a_i\mid q)+\alpha\,\pi_{\mathrm{unif}}(a_i\mid q)}.
  \label{eq:q3-wi}
\end{equation}

\paragraph{(ii) Limit $\alpha\to0$. \hfill\textnormal{[1]}}
As $\alpha\to0$, the mixture \eqref{eq:q3-mix} collapses to $\pi_{\mathrm{old}}$, so the weight tends to $1$ and the estimator reduces to GRPO:
\begin{equation}
  \alpha\to0:\quad \pi_{\mathrm{mix}}\to\pi_{\mathrm{old}}\ \Longrightarrow\ w_i\to1
  \ \Longrightarrow\ \hat g_{\mathrm{mix}}\to\hat g_{\mathrm{GRPO}}.
  \label{eq:q3-limit}
\end{equation}

\paragraph{(iii) The shifted group mean. \hfill\textnormal{[1+2]}}
(a) Since $V^{\pi}$ is linear in $\pi$ and $\pi_{\mathrm{mix}}$ is an $\alpha$-blend, the mixture value is the same blend of the component values:
\begin{equation}
  V^{\pi_{\mathrm{mix}}}(q)=(1-\alpha)\,V^{\pi_{\mathrm{old}}}(q)+\alpha\,V^{\pi_{\mathrm{unif}}}(q).
  \label{eq:q3-vmix}
\end{equation}
(b) The weights correct the score term but not the baseline: the plain group mean $\bar r$ averages
mixture draws, so it estimates the shifted $V^{\pi_{\mathrm{mix}}}$ rather than $V^{\pi_{\mathrm{old}}}$,
whereas the self-normalised weighted mean $\bar r_w$ recovers $V^{\pi_{\mathrm{old}}}$:
\begin{equation}
  \mathbb{E}_{\pi_{\mathrm{mix}}}[\bar r]=V^{\pi_{\mathrm{mix}}}(q)\neq V^{\pi_{\mathrm{old}}}(q),
  \qquad
  \bar r_w=\frac{\sum_{j} w_j\,r_j}{\sum_{j} w_j}\ \longrightarrow\ V^{\pi_{\mathrm{old}}}(q).
  \label{eq:q3-baseline}
\end{equation}
Re-centring on $\bar r_w$ gives the reweighted advantage that keeps $\hat g_{\mathrm{mix}}$ unbiased for the $\pi_{\mathrm{old}}$ gradient:
\begin{equation}
  \tilde A_i=\frac{r_i-\bar r_w}{\sigma_r}.
  \label{eq:q3-Atilde}
\end{equation}
Left uncorrected, this shift is a constant $\alpha\bigl(V^{\pi_{\mathrm{old}}}-V^{\pi_{\mathrm{unif}}}\bigr)/\sigma_r$ in
every advantage; it biases the gradient once $\pi_\theta\neq\pi_{\mathrm{old}}$ and inflates the variance
otherwise, so re-centring on $\bar r_w$ is necessary.

\paragraph{(iv) Variance comparison. \hfill\textnormal{[4]}}
Both are unbiased for the same $g$ (with $\tilde A$ from (iii)), so we compare second moments, with
$s=\nabla_\theta\log\pi_\theta(a\mid q)\,\hat A$:
\begin{equation}
  \mathrm{Var}(\hat g_{\mathrm{GRPO}})=\tfrac1K\mathrm{Var}_{\pi_{\mathrm{old}}}[s],\qquad
  \mathrm{Var}(\hat g_{\mathrm{mix}})=\tfrac1K\mathrm{Var}_{\pi_{\mathrm{mix}}}[w\,s].
  \label{eq:q3-var}
\end{equation}
Only the weight $w$ changes the GRPO second moment $\mathbb{E}_{\pi_{\mathrm{old}}}[s^2]$:
\begin{equation}
  \mathbb{E}_{\pi_{\mathrm{mix}}}\!\bigl[w^2 s^2\bigr]=\mathbb{E}_{\pi_{\mathrm{old}}}\!\bigl[w\,s^2\bigr],
  \qquad
  w=\frac{1}{(1-\alpha)+\alpha\,\pi_{\mathrm{unif}}/\pi_{\mathrm{old}}}\ \gtrless\ 1 .
  \label{eq:q3-var2}
\end{equation}
So mixing \emph{increases} the variance when $R$ puts large advantage where $\pi_{\mathrm{old}}$ is
large ($w>1$), and \emph{decreases} it where $\pi_{\mathrm{old}}$ is small ($w<1$, the uniform part
adding coverage).

\paragraph{(v) Weight collapse for long responses. \hfill\textnormal{[2+2]}}
Assume the per-token ratio $\pi_{\mathrm{old}}(a_{i,t}\mid q,a_{i,<t})/\pi_{\mathrm{mix}}(a_{i,t}\mid q,a_{i,<t})=c$ is constant, so the full-response weight is $w_i=c^{T}$ for $T$ tokens.
(a) The per-token weight has mean $1$ under $\pi_{\mathrm{mix}}$, so by Jensen
$\log c=\mathbb{E}_{\pi_{\mathrm{mix}}}[\log w_t]\le0$, i.e.\ $c\le1$ (strict for $\alpha>0$):
\begin{equation}
  w_i=c^{T}\xrightarrow{\,T\to\infty\,}0,\qquad
  \log c=\mathbb{E}_{\pi_{\mathrm{mix}}}[\log w_t]\;\le\;\log\mathbb{E}_{\pi_{\mathrm{mix}}}[w_t]=0 .
  \label{eq:q3-collapse}
\end{equation}
So $w_i=c^{T}\to0$: almost every response gets vanishing weight, the mean held at $1$ by rare outliers.
(b) Then one rollout dominates: the effective sample size collapses and $\hat g_{\mathrm{mix}}$ has
runaway variance for long $T$. Clipping then self-normalising the weights keeps $K_{\mathrm{eff}}$
bounded, at the cost of bias:
\begin{equation}
  K_{\mathrm{eff}}=\frac{\bigl(\sum_i w_i\bigr)^2}{\sum_i w_i^2}\xrightarrow{\,T\to\infty\,}1,
  \qquad
  \tilde w_i=\frac{\min(w_i,\tau)}{\sum_j \min(w_j,\tau)} .
  \label{eq:q3-ess}
\end{equation}

\paragraph{Walkthrough.}
The workflow composes two LangGraph state machines (Fig.~\ref{fig:partii-workflow}). The
\texttt{planning/} graph first produces a \texttt{Plan} through a propose--critique loop
(\texttt{planner}\,$\leftrightarrow$\,\texttt{plan\_reviewer}) bounded by \texttt{num\_rounds}, \emph{before}
any execution; the \texttt{adaptive\_planning/} graph then executes that plan one step at a time and is
where adaptation happens.
\emph{Trigger.} After each step, \texttt{execute\_step} records a summary and a structured outcome and
advances \texttt{step\_index}; \texttt{adaptive\_review} then reads that latest summary/outcome together
with the remaining steps and returns a boolean \texttt{needs\_adaptation}. When it is \texttt{True}
(on a fulfilled step), \texttt{route\_after\_review} routes to \texttt{replan\_tail} --- the revision.
\emph{What changes vs.\ what is fixed.} Only the not-yet-executed \emph{tail} is rewritten:
\texttt{replan\_tail} regenerates the remaining steps from the reviewer's recommendations and
\texttt{merge\_plan} splices \texttt{completed\_prefix} $+$ \texttt{new\_tail} back into \texttt{plan}.
The completed prefix and its outputs are held fixed (never re-proposed or re-run), as are the task and
the available-agent set; earlier steps are never revisited.
\emph{Termination.} \texttt{route\_after\_review} returns \texttt{END} when the last step was \emph{not}
fulfilled, or when the plan is exhausted (\texttt{step\_index}\,$>$\,$|\texttt{plan}|$); otherwise control
advances to the next step, or adapts and then advances. Every non-terminal cycle ends in an
\texttt{execute\_step} that increments \texttt{step\_index}, so the run makes monotone progress and stops
once the (possibly rewritten) plan is consumed or a step fails. Crucially, nothing caps the number of
adaptations or the length of the rewritten tail, so termination ultimately relies on the reviewer
eventually setting \texttt{needs\_adaptation} to \texttt{False}, with LangGraph's recursion limit as a
backstop.

% ---- II.2 ------------------------------------------------------------------
\subsection*{II.2\quad Problems (9 marks)}
\label{sec:partii-problems}
The two graphs are individually sound; the weaknesses are in how they compose into a closed loop, specifically:

\textbf{(P1) On failure the loop halts instead of re-planning, it is closed-loop only on \emph{success}.}
\texttt{route\_after\_review} (\texttt{adaptive\_planning/nodes.py}) tests whether the last
\texttt{step\_outcomes} entry is \texttt{fulfilled} \emph{before} it ever consults
\texttt{needs\_adaptation}: a non-fulfilled step is routed straight to \texttt{END}. The adaptive branch
therefore fires only when a step \emph{succeeds}, whereas a failed step that invalidates the rest of the plan 
aborts the run. This inverts the basic
closed-loop principle that a controller should react most strongly to \emph{adverse} feedback, and leaves
the ``adaptive'' planner brittle where a static plan would also fail.

\textbf{(P2) No bound on the number of adaptations or the plan length, termination is not guaranteed.}
Unlike the planning loop (\texttt{num\_rounds}) or \texttt{self\_debug} (\texttt{max\_n\_attempts}), the
adaptive loop carries \emph{no} counter. \texttt{replan\_tail} may return a tail \emph{longer} than the one
it replaces, and \texttt{adaptive\_history} is recorded but never read back to suppress a repeated
recommendation. Progress rests solely on \texttt{step\_index} eventually exceeding $|\texttt{plan}|$; but if
the reviewer keeps growing the tail, that target recedes each round, so the loop can oscillate
(re-proposing near-identical steps) or run away, caught only by LangGraph's default
\texttt{recursion\_limit}, which \emph{raises} rather than degrading.

\textbf{(P3) The adapt decision is made on impoverished, inconsistent observations.}
\texttt{adaptive\_review} sees only the \emph{last} step's free-text summary and its outcome dict, plus the
remaining steps; it never sees earlier steps, the raw artifacts, or the task rationale. The \emph{decision}
to adapt is thus \emph{less} informed than the \emph{action} it triggers: \texttt{replan\_tail} is handed
\emph{all} completed-step summaries. Judging whether the whole tail is still valid from one lossy,
LLM-written summary is weak observability of execution feedback, and invites both missed adaptations (an
issue that surfaced two steps earlier) and spurious ones.

\textbf{Further, briefly.} the completed prefix is frozen, so an early mistake revealed late can never be
undone; the reviewer$\to$planner hand-off is free-text \texttt{recommendations} with no check that the new
tail obeyed them (reviewer--planner coupling); and the structured-output calls have no error handling, so a
single parse failure crashes the run.

% ---- II.3 ------------------------------------------------------------------
\subsection*{II.3\quad Proposed improvements (9 marks)}
\label{sec:partii-improvements}
\textbf{(I1, for P1) Treat failure as a trigger, not a halt: bounded recovery re-plan.}
Modify \texttt{route\_after\_review} so that when a step is not fulfilled, it is redirected to a recovery 
invocation of \texttt{replan\_tail}. Initialize this recovery path with the failed step’s \texttt{reason} 
to enable the planner to adapt its strategy and bypass the issue. The recovery process should be constrained 
by a new \texttt{max\_recoveries} limit, after which execution aborts if no successful resolution is achieved. 
This restores
closed-loop behaviour on adverse feedback (the plan can repair itself) instead of discarding all completed
work; it costs extra LLM calls and latency, and risks thrashing on a genuinely impossible step, which the
budget bounds.

\textbf{(I2, for P2) Bound the loop explicitly.}
Add a \texttt{max\_adaptations} counter to \texttt{AdaptivePlanState} or \texttt{PlanContext} (mirroring
\texttt{num\_rounds}), reject a rewritten tail that exceeds the original length by more than a fixed slack,
and consult \texttt{adaptive\_history} to refuse a recommendation already tried; set an explicit
\texttt{recursion\_limit} as a hard backstop. Together these give a termination guarantee and remove the
oscillation mode. The trade-off is that a useful late adaptation can be cut off once the cap is hit, so the
cap should be configurable rather than hard-coded.

\textbf{(I3, for P3) Widen and align the observation, and close the hand-off.}
Give \texttt{adaptive\_review} the same context \texttt{replan\_tail} receives: the rolling list
of \emph{all} completed-step summaries together with the structured \texttt{step\_outcomes} (and, where
cheap, key artifacts), so the decision is as informed as the action. Make \texttt{recommendations} a
typed, itemised contract that \texttt{merge\_plan} checks the new tail against, so a reviewer request cannot
be silently ignored. This costs larger prompts (more tokens and latency) and a little extra plumbing, but
directly removes the blind-spot and the coupling gap behind P3.
